# helpCisco

Сайт сообщества по настройке **NAT в Cisco IOL** (IOS on Linux). Снаружи — обычный
гайд и лента, где люди выкладывают свои конфиги и обсуждают их. Внутри — скрытый функционал.

## Что внутри

1. **Фасад — гайд по NAT.** Пошаговая инструкция (роли интерфейсов, ACL, PAT/overload, проверка).
2. **Лента конфигов сообщества.** Люди публикуют свои настройки (форма «Поделись своим конфигом»),
   посты появляются в ленте. Под каждым постом — **чат-обсуждение** между посетителями.
3. **Общий живой чат.** Отдельная секция — общий чат для всех, кто сейчас на сайте.
   Чаты обновляются автоматически (опрос каждые ~3.5 сек), ник запоминается в браузере.
4. **ИИ-поддержка в углу.** Кнопка «?» справа снизу. Вместо обычной поддержки отвечает ИИ —
   на любой вопрос (по умолчанию — про NAT). Работает с реальной LLM (Anthropic/OpenAI) либо
   через встроенного оффлайн-ассистента, если ключ не задан.
5. **Скрытый раздел с мануалом** (`manual.php`). Не виден в навигации. Открывается только
   через скрытые кнопки или по секретному ключу. Контент добавим позже.

## Скрытые кнопки → скрытый мануал

В раздел `manual.php` ведут несколько незаметных триггеров (см. `script.js`):

| Способ | Действие |
|--------|----------|
| **Konami-код** | `↑ ↑ ↓ ↓ ← → ← → B A` на клавиатуре |
| **Логотип** | тройной клик по значку `®` рядом с «helpCisco» |
| **Подвал** | клик по версии `v2.4.1` с зажатым **Alt** |

Все они открывают `manual.php?key=<секрет>`. Кто пришёл без ключа и не активировал скрытую
кнопку — видит обычную страницу «404». Секрет задаётся в `.env` → `MANUAL_KEY`
(по умолчанию `ios-nat-2024`). После верного ключа доступ запоминается в сессии.

Сам мануал — заглушки (`Раздел 1/2/3`). Контент добавляется в `manual.php`, массив `$pages`.

## Настройки и секреты — файл `.env`

Все настройки и секреты (пароль админки, ключ мануала, ключ ИИ, модель) хранятся в файле
`.env` рядом с `config.php`. В репозиторий он **не коммитится** (см. `.gitignore`) — туда
попадают только боевые значения на твоём сервере. Шаблон со списком всех переменных —
`.env.example`.

```bash
cp .env.example .env   # создать свой .env
nano .env              # вписать пароль, ключ ИИ и т.д.
```

Переменные в `.env`:

| Переменная | Назначение |
|------------|------------|
| `SITE_NAME` | название сайта |
| `ADMIN_PASSWORD` | пароль модерации (`admin.php`) — **обязательно смени** |
| `MANUAL_KEY` | секретный ключ скрытого раздела `manual.php` |
| `AI_PROVIDER` | `openrouter` / `anthropic` / `openai` / `none` |
| `AI_API_KEY` | ключ провайдера ИИ (для OpenRouter — `sk-or-v1-...`) |
| `AI_MODEL` | модель (для OpenRouter — id вида `openai/gpt-4o-mini`) |
| `AI_SYSTEM_PROMPT` | системная подсказка ассистента (необязательно) |

**Формат файла.** Одна строка — одна переменная: `КЛЮЧ=значение`. Кавычки нужны только если в
значении есть пробелы (`AI_SYSTEM_PROMPT="Ты ассистент ..."`). Строки, начинающиеся с `#`, —
комментарии. Пустой `AI_API_KEY=` означает «ключа нет» → включается оффлайн-ассистент.

**Приоритет источников:** переменные окружения процесса → `.env` → значения по умолчанию.
Секрет можно задать и через окружение (например, systemd `EnvironmentFile=/etc/helpcisco.env`) —
оно перекроет `.env`.

**После правки `.env` перезапусти сайт**, чтобы новые значения применились:
- systemd: `systemctl restart helpcisco`;
- локальный запуск: останови `php -S` (Ctrl+C) и запусти снова.

**Безопасность.** `.env` хранит боевые секреты, поэтому он **не коммитится** в git и на сервере
ставится в режим `600` (читает только владелец). Никогда не вставляй настоящий ключ в `config.php`
или другой файл, который попадёт в репозиторий.

## ИИ-поддержка: бесплатная модель через OpenRouter (пошагово)

[OpenRouter](https://openrouter.ai) — единый шлюз к десяткам ИИ-моделей: один ключ, один API
(совместим с OpenAI). Среди моделей есть **бесплатные** — их id заканчивается на `:free`.
По умолчанию (`AI_API_KEY` пуст) сайт отвечает встроенным оффлайн-ассистентом по NAT; ниже —
как подключить реальную бесплатную модель.

### Шаг 1. Получить ключ OpenRouter
1. Зайди на <https://openrouter.ai> и войди (Google / GitHub / email).
2. Открой **Keys** → **Create Key**, задай имя, нажми создать.
3. Скопируй ключ вида `sk-or-v1-...` — он показывается **один раз**.
4. (Рекомендуется) на ключе можно выставить лимит расходов (*Credit limit*) — для бесплатных
   моделей можно `0`, чтобы случайно не уйти в платные.

### Шаг 2. Выбрать бесплатную модель
1. Открой каталог, отфильтрованный по нулевой цене:
   <https://openrouter.ai/models?max_price=0>
2. Кликни по модели и скопируй её **id** — он вида `provider/model:free`.
3. Примеры бесплатных моделей (список меняется — сверяйся со страницей выше):
   - `deepseek/deepseek-chat-v3-0324:free`
   - `meta-llama/llama-3.3-70b-instruct:free`
   - `google/gemini-2.0-flash-exp:free`
   - `qwen/qwen-2.5-72b-instruct:free`
4. Нюансы бесплатных моделей: есть ограничение по частоте запросов (rate limit), а для части
   моделей нужно один раз разрешить обработку данных в **Settings → Privacy** на OpenRouter.

### Шаг 3. Прописать в `.env`
```ini
AI_PROVIDER=openrouter
AI_MODEL=deepseek/deepseek-chat-v3-0324:free
AI_API_KEY=sk-or-v1-вставь_сюда_свой_ключ
```
Модель можно менять в любой момент — просто поправь `AI_MODEL` на другой id.

### Шаг 4. Применить
- systemd: `systemctl restart helpcisco`
- локально: перезапусти `php -S` (Ctrl+C → снова `php -S localhost:8000 router.php`).

Альтернатива для systemd — положить ключ в окружение (перекрывает `.env`):
```bash
echo 'AI_API_KEY=sk-or-v1-...' >> /etc/helpcisco.env
systemctl restart helpcisco
```

### Шаг 5. Проверить, что отвечает именно модель
Нажми кнопку **«?»** в правом нижнем углу сайта и задай вопрос. Чтобы убедиться наверняка,
дёрни эндпоинт напрямую и посмотри поле `source`:
```bash
curl -s -X POST http://localhost:8000/ai.php \
  -H 'Content-Type: application/json' \
  -d '{"message":"привет"}'
```
- `"source":"openrouter"` — отвечает реальная модель ✅
- `"source":"offline"` — сработал запасной оффлайн-ассистент (значит, запрос к OpenRouter не прошёл).

### Если отвечает оффлайн-ассистент (`source: offline`)
Оффлайн-ассистент включается, когда запрос к OpenRouter не удался. Частые причины:
- `AI_PROVIDER` не `openrouter` или `AI_API_KEY` пуст;
- неверный или отозванный ключ;
- опечатка в `AI_MODEL` либо модель сняли с раздачи — возьми актуальный id со страницы моделей;
- для выбранной `:free` модели не включено разрешение в **Settings → Privacy** на OpenRouter;
- упёрся в бесплатный лимит запросов — подожди или выбери другую `:free` модель;
- на сервере нет расширения **php-curl** (нужно для запроса) — поставь `apt install php-curl`
  и перезапусти сервис.

### Платные модели
Шаги те же: пополни баланс в **Credits** на OpenRouter и укажи в `AI_MODEL` платный id
(без суффикса `:free`), например `openai/gpt-4o-mini` или `anthropic/claude-3.5-sonnet`.

Системную подсказку ассистента можно переопределить в `.env` → `AI_SYSTEM_PROMPT`
(по умолчанию — про NAT в Cisco IOL).

## Как хранятся данные

База данных не нужна — всё в JSON-файлах в `data/` (запись с блокировкой файла):

- `data/posts.json` — опубликованные конфиги сообщества;
- `data/chat.json` — сообщения чатов по комнатам (`global` и `post-<id>`).

В репозитории лежит демо-контент, чтобы сайт сразу выглядел живым.

## Установка по IP одной командой (Debian/Ubuntu)

`install.sh` поднимает сайт как systemd-сервис `helpcisco` на отдельном порту (по умолчанию `8080`)
через встроенный PHP-сервер — не конфликтует с nginx/Apache/FastPanel на 80/443.

```bash
ssh root@СЕРВЕР
apt update && apt install -y git
git clone <ссылка_на_репозиторий> helpcisco
cd helpcisco
chmod +x install.sh
sudo ./install.sh            # порт 8080 (или ближайший свободный)
# свой порт:  sudo ./install.sh 8090
```

После установки:

- Сайт: `http://IP:ПОРТ/`
- Модерация: `http://IP:ПОРТ/admin.php`
- **Смени** `ADMIN_PASSWORD` и `MANUAL_KEY` в `/var/www/helpcisco/.env` (его создаёт установщик
  из шаблона), затем `systemctl restart helpcisco`.

Управление: `systemctl status|restart helpcisco`, логи: `journalctl -u helpcisco -f`.

## Локальный запуск (нужен PHP)

```bash
php -S localhost:8000 router.php
# сайт:      http://localhost:8000/
# модерация: http://localhost:8000/admin.php
```

## Перезагрузка и проверка сайта — `restart.sh`

`restart.sh` перезапускает сайт и проверяет, что работают **все** функции, включая то,
**реально ли подключён и отвечает ИИ**.

```bash
chmod +x restart.sh
./restart.sh                 # перезапуск + полная проверка
./restart.sh --no-restart    # только проверки, без перезапуска
./restart.sh --url http://IP:8080   # проверить конкретный адрес
./restart.sh --port 8090     # задать порт
```

Скрипт сам определяет режим: если есть systemd-сервис `helpcisco` — перезапускает его и берёт порт
из юнита; иначе перезапускает локальный `php -S`. Что проверяется:

- главная страница и статика (`style.css`, `script.js`, `theme.js`);
- лента конфигов (`posts.php`) и чат (`chat.php`), отклонение битой комнаты;
- скрытый мануал: без ключа — «404», с ключом из `.env` — доступ открывается;
- защита служебных файлов: `config.php`, `lib.php`, `.env`, `data/` отдают `403`;
- админка отдаёт форму входа;
- **ИИ**: шлёт запрос в `ai.php` и смотрит поле `source`. Если в `.env` задан провайдер и ключ,
  но ответил `offline` — это **провал** (ключ/модель/сеть не работают, скрипт подскажет, что
  проверить). Если ключ не задан — оффлайн-ассистент считается нормой.

Код выхода `0` — всё работает, `1` — есть проваленные проверки (удобно для cron/CI).

## Файлы

| Файл | Назначение |
|------|------------|
| `index.php` | Главная: гайд, лента конфигов, чаты, форма публикации, ИИ-поддержка, скрытые кнопки |
| `style.css` | Стили (тёмная тех-тема) |
| `script.js` | Фронтенд: чаты, публикация, ИИ-виджет, скрытые триггеры |
| `theme.js` | Случайная цветовая тема на каждую загрузку (подключается в `<head>`) |
| `chat.php` | Бэкенд чата (комнаты: `global`, `post-<id>`) |
| `posts.php` | Публикация и выдача конфигов сообщества |
| `ai.php` | ИИ-поддержка: LLM (Anthropic/OpenAI) или оффлайн-ассистент |
| `manual.php` | Скрытый раздел с мануалом (по секретному ключу) |
| `admin.php` | Модерация: посты и чаты, удаление/очистка |
| `lib.php` | Общие помощники (JSON-хранилище с блокировкой, санитизация) |
| `config.php` | Загружает `.env` и задаёт настройки (с дефолтами) |
| `.env` / `.env.example` | Секреты (пароль, ключ мануала, ключ ИИ); `.env` не в git |
| `router.php` | Роутер встроенного PHP-сервера: закрывает `data/`, `config.php`, `lib.php`, `.env` |
| `install.sh` | Установка по IP: PHP + systemd-сервис на отдельном порту |
| `restart.sh` | Перезапуск сайта + проверка всех функций (включая работу ИИ) |
| `data/` | JSON-хранилище (закрыто от прямого доступа) |

## Безопасность

- Секреты лежат в `.env` (пароль `ADMIN_PASSWORD`, ключ мануала `MANUAL_KEY`, ключ ИИ `AI_API_KEY`) —
  **обязательно поменяй** дефолты перед боем. Файл `.env` не коммитится в git.
- Прямой доступ к `data/`, `config.php`, `lib.php` и `.env` закрыт `router.php` (встроенный сервер)
  и `data/.htaccess` (Apache).
- Вход по IP идёт по `http://`. Для личного использования это норм; нужен HTTPS — потребуется домен.
