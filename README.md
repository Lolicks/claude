# Тест для тебя 💖

Милый интерактивный сайт-тест. Она проходит тест — её ответы **отправляются на сервер**, а ты заходишь в защищённую паролем админку и видишь все её попытки: сколько раз проходила и какие ответы были в каждой.

## Как это работает

```
Она:  index.html  →  отвечает на вопросы  →  ответы летят на сервер (save.php)
                                                       │
                                                       ▼
                                          data/attempts.json (все попытки)
                                                       │
Ты:   admin.php  ←  вводишь пароль  ←──────────────────┘
```

- Каждое прохождение теста = **отдельная попытка** со своей датой и временем. Попытки не накладываются друг на друга, даже если пройдены в разное время.
- Данные хранятся на сервере в файле `data/attempts.json` (база данных не нужна).

## 1 часть — тест (`index.html`)

- Первый вопрос: «Ты меня любишь?» с кнопками **Да** и **Нет**. Кнопку **Нет** нажать нельзя — она убегает 😏
- Ещё 5 вопросов с вариантами ответов:
  1. Что ты почувствовала, когда мы познакомились?
  2. Чем тебе больше всего нравится заниматься вместе?
  3. Как сильно ты по мне скучаешь?
  4. Куда бы ты хотела отправиться со мной?
  5. Сколько сердечек я заслужил?
- Прогресс-бар показывает, сколько пройдено. В конце ответы отправляются на сервер.

## 2 часть — админка (`admin.php`)

- Открываешь `admin.php`, вводишь **пароль**.
- Видишь все попытки (новые сверху): номер попытки, дату/время и все ответы.
- Сверху — счётчик общего числа попыток.
- Кнопки «Очистить всё» и «Выйти».

## 🚀 Установка по IP одной командой (Debian/Ubuntu)

Скрипт `install.sh` поднимает сайт как systemd-сервис на **отдельном порту** (по умолчанию `8080`) через встроенный PHP-сервер. Заходишь по `http://IP_СЕРВЕРА:ПОРТ/`. Домен не нужен.

**Почему отдельный порт:** на сервере порты 80/443 часто уже заняты другим веб-сервером (nginx, Apache, панель вроде FastPanel). Скрипт с ними не конфликтует — берёт свободный порт и работает рядом.

### Запуск

```bash
# 1. Подключись к серверу
ssh root@5.101.119.187

# 2. Скачай проект
apt update && apt install -y git
git clone <ссылка_на_репозиторий> love-test
cd love-test

# 3. Установи (по умолчанию порт 8080)
chmod +x install.sh
sudo ./install.sh
# или свой порт:  sudo ./install.sh 8090
```

### Что сделает скрипт

1. Установит PHP (`php-cli`, `php-mbstring`), если его нет.
2. Подберёт свободный порт (если 8080 занят — возьмёт следующий).
3. Скопирует сайт в `/var/www/love-test` (без служебных файлов и чужих ответов).
4. Выставит права и сделает папку `data/` доступной для записи.
5. Создаст и запустит systemd-сервис `love-test` (автозапуск при перезагрузке).
6. Откроет нужный порт в `ufw` (если включён).
7. Покажет ссылки с твоим IP и портом.

### После установки

- Тест (для неё):  `http://IP_СЕРВЕРА:ПОРТ/`
- Админка (для тебя): `http://IP_СЕРВЕРА:ПОРТ/admin.php`
- **Поменяй пароль** админки:
  ```bash
  nano /var/www/love-test/config.php   # замени 'changeme' на свой пароль
  systemctl restart love-test
  ```

### Управление сервисом

```bash
systemctl status love-test     # статус
systemctl restart love-test    # перезапуск (после правки файлов)
journalctl -u love-test -f     # живые логи
```

> Если снаружи сайт не открывается, а сервис `active` — значит порт закрыт **внешним фаерволом хостинга**. Открой выбранный порт (например 8080) в панели/настройках безопасности провайдера.

## 🔒 Безопасность

- Пароль админки лежит в `config.php` — **обязательно поменяй** значение по умолчанию `changeme`.
- Прямой доступ к `data/`, `config.php` и служебным файлам закрыт роутером `router.php` (для встроенного PHP-сервера) и файлом `data/.htaccess` (для Apache-хостинга).
- Реальные ответы (`data/attempts.json`) не попадают в репозиторий — см. `.gitignore`.
- Вход по IP идёт по `http://` (без шифрования). Для маленького личного теста это норм; если нужен HTTPS — понадобится домен, скажи, и добавлю установку с SSL.

## Файлы

| Файл | Назначение |
|------|------------|
| `index.html` | Страница с тестом |
| `script.js` | Логика теста: вопросы, убегающая кнопка, отправка ответов, анимации |
| `hearts.js` | Плавающие сердечки на фоне (фон теста и админки) |
| `style.css` | Современные стили теста и админки |
| `save.php` | Принимает ответы и сохраняет попытку в `data/attempts.json` |
| `admin.php` | Админ-страница с паролем: показывает все попытки |
| `config.php` | Настройки: пароль админки и путь к файлу данных |
| `router.php` | Роутер встроенного PHP-сервера: закрывает доступ к `data/` и `config.php` |
| `install.sh` | Установка по IP: PHP, systemd-сервис на отдельном порту |
| `data/` | Хранилище попыток (закрыто от прямого доступа) |

## Локальный запуск (для проверки)

Нужен PHP:

```bash
php -S localhost:8000 router.php
# тест:    http://localhost:8000/
# админка: http://localhost:8000/admin.php
```
